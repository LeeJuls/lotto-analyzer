## 변경 사항 및 의사결정 내역
1. **문제 상황 지속 및 요구사항 변경**
   - 로컬 환경의 파이썬 업데이터 스크립트는 매우 잘 동작하나, 사용자가 웹사이트 상("인터넷에서도 볼 수 있게")에서 직접 버튼을 클릭해 실시간으로 1213회차 등을 업데이트하기를 원함.
2. **의사 결정**
   - 파이썬으로 구현한 '네이버 검색결과 HTML 크롤링 후 당첨번호 파싱' 로직을 순수 Javascript 코드(`DOMParser` 활용)로 번역.
   - 단, 브라우저 단에서는 네이버 서버 접속 시 CORS 에러가 발생하므로 `api.codetabs.com/v1/proxy` 퍼블릭 프록시 서비스를 브릿지로 이용함. 
3. **작업 내역**
   - `index.html` 내의 `fetchDraw` 함수를 새롭게 재구현함.
   - 알림으로 대체했던 "데이터 갱신 안내" 버튼을 기존의 `refreshData`가 즉시 실행되는 "데이터 갱신" 버튼으로 롤백.
   - `index.html`만 단독으로 서버(Github Pages/Render 등)에 배포되어도 최신 데이터 갱신이 가능하도록 완벽히 로직 분리 완료.
